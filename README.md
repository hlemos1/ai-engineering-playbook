# AI Engineering Playbook

Como construir sistemas reais usando IA como camada de execucao.

---

## Tese

IA nao substitui engenharia.
IA amplifica engenharia — se houver arquitetura.

A maioria falha porque usa IA sem estrutura.
Aqui esta o modelo que uso para sair de ideia para producao com consistencia.

---

## Modelo operacional

1. Problema de negocio
2. Arquitetura antes do codigo
3. Stack definida
4. IA como executor
5. Testes como validacao
6. CI/CD como disciplina
7. Deploy e monitoramento

---

## Papel humano vs IA

### Humano (arquiteto)
- Definicao do problema
- Desenho de arquitetura
- Decisoes tecnicas
- Validacao de qualidade

### IA (executor)
- Geracao de codigo
- Criacao de testes
- Refatoracao
- Documentacao

---

## Regra de ouro

Se voce nao sabe explicar a arquitetura, nao use IA.
Voce so vai gerar bagunca mais rapido.
