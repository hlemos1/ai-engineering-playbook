# Contributing

Thanks for your interest in contributing to the AI Engineering Playbook.

## How to contribute

### Content improvements
- Fix typos, grammar, or unclear explanations
- Add context to existing sections
- Suggest better examples

### New content
- New case studies following the existing format (Problem -> Solution -> Architecture -> Results -> Lessons)
- Additional prompt patterns with real-world context
- New common mistakes with fixes

### Translations
- Translations to other languages are welcome
- Place translated files in a `i18n/[lang]/` directory

## Process

1. Fork the repository
2. Create a branch (`git checkout -b improve-xyz`)
3. Make your changes
4. Submit a pull request with a clear description of what changed and why

## Guidelines

- Keep the same opinionated, direct tone
- No filler content — every sentence should add value
- Include real numbers and examples when possible
- If adding a case study, follow the exact structure of existing ones

## What we don't accept

- Generic AI advice without practical backing
- Tool-specific tutorials (this is about methodology, not tools)
- Content that contradicts the core thesis (AI amplifies engineering, doesn't replace it)
